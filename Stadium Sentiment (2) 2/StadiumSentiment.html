Sentiment Analysis: Public Views Towards New Stadiums

By: Sasha Novitsky and Afzal Patel

tl;dr

Within the last few decades, new sports stadiums seem to be popping up every day. While many of these stadiums are financed privately by individuals and corporations, the majority are financed by a mix of public and private funds.[1] But what do the citizens really think of these costly stadiums? Using a large twitter database[2], this project explores the relationship between public sentiment of the ten largest stadiums under construction in the United States (as of late 2019) and the months until completion, and attempts to build a model that predict the average public sentiment towards a stadium given the specific number of months away from completion.

Overview

A dataset of multiple billion tweets was parsed using an algorithm written in Python that looked for specific, well thought-out keywords related to the ten largest stadiums under construction in the United States. The idea gathering the dataset was to extract relevant tweets, and since most individuals are not actively tweeting about stadiums nor their ongoing construction, we sought to have our data be a mixture of tweets relating to the stadiums and teams, and tweets that contain terms that may be found opinionated statements[3].

Specifically taken into consideration while crafting the various keyword lists were minimalism[4], official handles for the organization(s) tied to the stadium, team and stadium names, appropriate slogans, and common hashtags found on twitter when searching for these stadiums on the app. Keywords were also uppercased and checked whether they were contained[5] in the tweet’s uppercased text. While parsing the original dataset, this eliminates the need to align casing between keywords and potential matches in the tweet’s text and the documentation for Python’s string library provides methods to handle casing. With this method, there was a possibility of multiple keywords appearing in a single tweet, creating duplicates. Duplicate data was handled as we cleaned our collection of tweets from the original dataset.

While each individual tweet in the original dataset contained over twenty attributes, the dataset this project uses focuses on text[6], place and time of creation, and language. After finding all of the valid[7] tweets containing the necessary keywords[8], they were saved into a file. Finally, the dataset was narrowed down[9] to about two-hundred thousand tweets. Notably, the dataset is further broken down and analysed in a later section of this project.

The following figure lists the ten American stadiums of focus in the order of projected capacity and provides other important qualities. Unless stated otherwise, affiliated organizations and sports are assumed to be professional.

Figure 1. Stadium Traits

After gathering the data, sentiment analysis was performed using the Python library “TextBlob”. The sentiment analysis reads the text of each tweet and returns the ‘sentiment’, which is defined as “a named tuple of the form sentiment(polarity, subjectivity).” [10] Polarity ranges from [-1.00, 1.00], where (-1) means the text speaks very negatively and (+1) means the text speaks very positively. Subjectivity ranges from [0.00, 1.00], where (0) means the text displays no subjectivity and (+1) means the text is entirely subjective.

Training a Model

        Training a model on this dataset turned out to be a challenge. The data is highly non-linear and never really trends in any specific direction much over the time period, but we still attempted. Because we wanted to observe a relationship between the sentiment of tweets (two variables) and the time until completion, we chose to make the scheduled month of completion “month 0” and count backwards from there. This became our main horizontal axis for all of the following graphs. Because Globe Life Field is scheduled for completion in March, 2020, that month became “month 0,” December of 2019  became “month -3,” December of 2018 became “month -15” and so on. Below are the results of our first attempt at a classic linear ordinary least squares style regression with only one independent variable. This analysis was performed using Python’s seaborn library.[13] 

Figure 7A. Regression of Polarity on Months until Completion

Figure 7B. Regression of Subjectivity on Months until Completion

        This simple model clearly fit the data very poorly. The Polarity model had an R-Squared of only (0.0000720) while the Subjectivity model’s R-Squared value was slightly higher at (0.0005078). Both of these models are extremely terrible, as both fit something on the order of a few hundredths of a percent of their respective datasets.

        To help visualize the inaccuracy of this model, Figures 8A and 8B are scatter plots of the residuals from the regression analysis plots in Figures 7A and 7B. Because we had no idea what the result may be, a single independent variable linear regression was our first attempt at building a model to fit our data.

Figure 8A. Residuals of Polarity on Month Regression

Figure 8B. Residuals of Subjectivity on Month Regression

        The residuals of the model are defined as “the vertical distance from the data point and the regression line.”[14] Because our residuals look almost exactly like the above Figures 7A and 7B without the regression line through them, this suggests that our model fits the data very, very poorly. However, there is a small silver lining - while the model leaves very high residuals, it is clearly unbiased as a result of the consistent large range for all of the independent axis values. If our model was biased, the residuals would either converge or diverge, suggesting that the model only fit for either very large or very small values of “Months from Completion.”

The lasso model stays the worst, having a terrible R-squared and a root mean squared error significantly higher than either of the other two.

        Finally, we will discuss some conclusions of our models and results of the models.

Conclusions

         As we have discovered, predicting the sentiment towards the construction of stadiums is a challenging problem. Our initial attempt gave very poor results, and suggested that the number of months until completion does not greatly affect the sentiment of tweets towards that stadium. Our residual plots (Figures 8A and 8B) confirm this, as they show that the model consistently leaves very large residuals. Because of this poor result, we tried new models.

        These new models all took two independent variables to predict either the Polarity or Subjectivity of a tweet. The results of this were incredibly superior, as they gave us R-squared’s that fit our data hundreds of times better than the first attempt. After trying linear, ridge and lasso models, the linear appeared to be the best type of model to use for this dataset.

        While this appears to be a very positive result, we must look more at this type of model. First, what did our error consist of? Why did this model not fit the data better? The majority of our error came from estimation error, which is essentially that the regression line does a very poor job of predicting the actual data. This error is a very challenging problem to solve, as the data is consistently very broad and does not really trend in any direction. As a result, we attempted to mitigate this error with these second round attempts at building a model by trying to connect polarity and subjectivity to each other.

        While this attempt at improving the model did in fact improve it, it also makes it very hard to implement outside of past data because one needs either the specific sentiment or polarity at the given month. The easiest way to solve this problem would be to find a range of answers for a given time period with a range of either polarity or subjectities (depending on which you are attempting to find). As a result, this model could give you a possible range for either subjectivity or polarity. Again, you must remember that this model does not fit the data perfectly, and there is still a very high chance that the output of the model would not fit the period that you’re trying to project the model into very well.

        Overall, our model is a poor predictor of future public sentiment towards stadiums under construction. As stated before, this mostly comes from the sentiment being very broad and not changing much over time, which suggests that there is very little correlation between these two variables. If one were in the business of constructing stadiums, they should not expect sentiment towards their stadium to drastically change as the projected completion date approaches.

Technical Appendix

Throughout the project, the data was saved in CSV files that were accessible using Python script or Microsoft Excel. Many of the summary tables for the data in the “General Findings” and “Globe Life Field’s Data Findings” sections were performed using Excel. After this section, every other figure was generated using Python script on the same CSV files. For all models, the data was broken up into a training set and a test set, where the training set consisted of 13,800 data points and the test set consisted of the other 9,850 data points.

For our basic one-dimensional linear regression, performed this regression in two different ways. In the first, we used Pythons Pandas[16] library to shape the data into the necessary arrays, followed by using SkLearn’s linear regression to actually build the model. Because it was a very simple model, there was no optimization done on the model after the data had been modified into the correct, usable format. Secondly, we used the Seaborn (mentioned earlier) library to actually construct a visual of this model. We used the same data, and both libraries gave us the same results for this simple regression. The residual plots were performed using the Seaborn library as well.

When we moved onto the more complicated regressions, we moved away from the Pandas way of building these models and only used SkLearn. First, to visualize the three-dimensional data, we used Python’s MatPlotLib’s feature of “mplot3d.”[17] Since MatPlotLib is only intended for use with two dimensional graphs, the functions relating to three dimensional graphs are very limited. As a result, these graphs were not nearly as well designed as we would have liked, but we did not have the functionality to improve them.

The next three models were all built SkLearn’s linear model. The true linear model could not really be optimized, so we did not modify this command at all. The second model was SkLearn’s ridge regression. Because ridge regression takes an ‘alpha’ term as a parameter, we optimized this alpha term to minimize the mean squared error of the model on the test data. Figures 12A and 12B are visuals of this optimization.

Figure 12A. Polarity Alpha Ridge Optimization

Figure 12B. Subjectivity Alpha Ridge Optimization

        After finding this optimized alpha, a ridge model was trained on the data. Finally, we performed this same method for the lasso models that performed poorly in Figures 10A, 10B, 11A and 11B.

        After performing all of these regressions, SkLearn’s ‘metrics’ library was used to find the R-squared values and the Root Mean Squared Error terms for every model. These were then saved, transferred to an Excel document and graphed against each other for the comparison graphs shown in Figures 10A, 10B, 11A and 11B.
